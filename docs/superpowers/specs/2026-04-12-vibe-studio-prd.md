# VibeStudio PRD

**产品名称**: VibeStudio
**日期**: 2026-04-12
**状态**: 草稿
**定位**: 个人创作伴侣 — 专为有故事想讲的人设计的长篇小说 → 剧集化视频 AI 辅助工具

---

## 一句话定位

你的私人创作伙伴 — 写小说，拆剧集，AI 帮你把脑海里的故事变成视频。

---

## 核心定位

- **目标用户**：本人（个人使用），有长篇小说创作需求，脑海中有故事想转化为视频
- **数据策略**：核心数据本地，生成任务全走云端 API，不部署本地模型
- **工作流风格**：项目管理清晰感，固定线性流程，减少决策疲劳

## 核心设计原则

1. **固定线性流程** — 小说 → 知识库 → 剧集，不绕路，不纠结下一步
2. **小说与剧集双重管理** — 小说侧专注创作，剧集侧专注制作，知识库作为桥梁
3. **知识库实时同步创作上下文** — 续写时 AI 提供伏笔回顾、人物关系、待解悬念
4. **版本控制轻量化** — MD 层用 git，知识库层每次同步自动快照
5. **全AI配音** — 不依赖本地录音，云端语音合成

## 模块一：小说创作侧

### 1.1 MD 编辑器

- 每章独立 .md 文件
- 全屏沉浸式编辑体验
- 左侧章节列表（按序排列）
- 右侧 LLM 审阅面板（实时）

### 1.2 LLM 审阅面板

- **实时批注**：对当前章节内容做行内/段内批注
- **变更追踪**：识别并高亮本章新增角色、物品、情节变动
- **质量建议**：叙事节奏、对话优化、伏笔提示
- **关键信息提取**：场景描述、情绪标签、角色动作

### 1.3 知识库同步（手动触发）

每次手动触发后，LLM 同时输出两件事：

**知识库更新：**
- 新增角色 → 角色库（生成角色卡含形象描述）
- 物品变动 → 物品库（记录流转）
- 地点出现 → 地点库
- 情节推进 → 情节线（主线/支线/伏笔标记）
- 关系变化 → 关系网更新
- 情绪弧线 → 情感弧线更新

**剧集素材抽取：**
- 本章关键场景列表（AI 自动识别）
- 每个场景含：画面描述 + 对白摘要 + 情绪标签 + 场景类型
- 场景快照永久关联对应 MD 章节

### 1.4 续写入口

**主要入口：从剧情节点进入续写**
- AI 列出本章末尾的钩子、伏笔、待解悬念
- 显示与当前章节相关的角色/物品状态
- 你接着写，LLM 实时辅助

## 模块二：知识库

### 2.1 角色库

- **角色卡**：形象描述 + 关键性格标签 + 常用表情/动作
- **关系网**：角色间的关系、冲突、联盟（可视化图谱）
- **跨集一致性**：版本快照 + 强制引用模式

### 2.2 物品库

- 关键物品描述
- 物品流转历史（哪章出现，谁拿着，有什么象征意义）
- 物品状态追踪

### 2.3 地点库

- 场景地点列表
- 场景氛围描述
- 跨项目可复用

### 2.4 情节线

- **主线**：核心叙事弧
- **支线**：分支剧情
- **伏笔**：埋下的线索及对应章节
- **悬念**：待解问题列表
- **情感弧线**：角色情绪变化轨迹

### 2.5 全局搜索

- **全文搜索**：角色名、物品名、关键词，一键跳转到对应 MD 章节或场景
- **关系图谱**：角色关系网、情节线可视化，点击节点跳转

### 2.6 版本控制

- **MD 层**：git（现有）
- **知识库层**：每次"同步知识库"自动生成快照，可追溯可回滚
- **快照粒度**：条目级别（每个角色/物品/情节独立快照）

## 模块三：剧集制作侧

### 3.1 看板系统

**双视图切换：**

**集视图：**
```
┌─────────┬─────────┬─────────┬─────────┬─────────┐
│  剧本    │  分镜    │  Takes  │  粗剪   │  成片   │
├─────────┼─────────┼─────────┼─────────┼─────────┤
│ 第1集   │  ●○○   │  ○○○   │  ○○○   │  ○○○   │
│ 第2集   │  ○○○   │  ○○○   │  ○○○   │  ○○○   │
│ ...     │         │         │         │         │
└─────────┴─────────┴─────────┴─────────┴─────────┘
 ● = 已完成  ○ = 未完成
```

**场景视图：**
- 每个场景一行卡片
- 可按集筛选，也可跨集查看同一场景的所有 Takes 版本

### 3.2 AI 建议 + 确认机制

**集/场景拆分（手动触发）：**
- AI 根据章节长度、剧情节点建议分集方案
- 你确认或调整每集包含的章节/场景
- 场景由 AI 自动从已抽取的素材中提取，你调整归属

### 3.3 分镜管理

- **颗粒度**：脚本级（镜头 + 对白 + 音效 + 时长）
- **构图关键帧**：每个镜头可指定参考图（角色/场景/动作）作为锚点
- AI 生成分镜建议，你逐个确认或调整
- 可视化分镜面板

### 3.4 Takes 系统

- 每个镜头 AI 生成 3-5 个 Takes 版本（不同风格/构图/表演）
- **构图关键帧锚定**：角色卡 + 关键帧双保险，确保角色一致性
- 版本并排展示，你选最优版本进入剪辑线
- 觉得不好可要求 AI 重新生成

### 3.5 关键帧管理

**双关键帧系统：**
- **构图关键帧**（生成用）：锁定角色/场景/动作参考
- **时间线关键帧**（剪辑用）：标记镜头衔接点，控制节奏

### 3.6 角色一致性管理

**混合模式（角色卡 + 关键帧）：**
- 知识库角色卡（形象描述）作为生成 prompt 的一部分
- 构图关键帧辅助校准
- 首个镜头生成后锁定角色外观，后续镜头以此为参考

### 3.7 音频管线

- **全AI配音**：对白、旁白全部云端语音合成
- 背景音乐（BGM）：AI 推荐 + 你确认
- 音效（Foley）：AI 生成
- 音视频同步管理

### 3.8 剪辑合成

- AI 指导下的专业剪辑
- 多轨道、时间线编辑
- 时间线关键帧控制镜头衔接和节奏
- 本地完成

### 3.9 导出

- **MOV（ProRes）** — 高质量，最终归档
- MP4（H.264）— 日常预览可选

## 固定线性流程

```
小说MD编写
    ↓
[手动触发同步]
    ↓
┌─────────────────────┐
│ LLM分析输出：        │
│ 1. 知识库更新        │
│ 2. 剧集素材抽取      │
└─────────────────────┘
    ↓
[续写入口：剧情节点]
    ↓
小说继续创作 ←→ 知识库实时支撑
    ↓
[AI建议+确认] → 集/场景拆分
    ↓
看板推进：
  剧本 → 分镜（脚本级） → Takes（AI批量生成） → 粗剪 → 成片
         ↑                              ↑
         构图关键帧锚定               角色一致性混合模式
    ↓
音频：全AI配音
    ↓
导出：MOV（ProRes）
```

## 版本控制体系

| 层级 | 工具 | 机制 |
|------|------|------|
| MD 文件 | git | 每次保存自动提交历史 |
| 知识库 | 内置快照 | 每次同步自动生成快照，可回滚 |
| Takes 版本 | 内置版本 | 每个镜头的多个版本永久保留 |

## UI 哲学

- **项目管理清晰感**：看板结构让进度一目了然
- **固定流程减少决策**：不用想下一步该干嘛
- **上下文导航**：从剧情节点切入创作，知识库实时支撑一致性
- **极简设计**：只让用户关注自己需要的信息
- **全屏沉浸**：创作内容最大化，面板按需浮现

## 技术说明

- **图像/视频生成**: 云端 API
- **音频生成**: 云端语音合成 API
- **剪辑合成**: 本地完成
